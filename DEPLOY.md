# Инструкция по запуску и деплою

## Локальный запуск

1. Установите зависимости:
```bash
npm install
```

2. Запустите сервер:
```bash
npm start
```

Или напрямую:
```bash
node server.js
```

3. Откройте в браузере:
```
http://localhost:3100/implementation_schedule.html
```

## Деплой в интернет (Render, Railway, Fly.io и т.д.)

### Что нужно для деплоя:

1. **Файлы проекта:**
   - `server.js` - сервер Node.js/Express
   - `package.json` - зависимости и скрипты
   - `implementation_schedule.html` - главная страница
   - `xlsx.min.js` - библиотека для экспорта в Excel
   - `favicon.png` - иконка сайта

2. **Настройки платформы:**
   - **Start Command:** `npm start` или `node server.js`
   - **Port:** сервер автоматически использует переменную окружения `PORT` (по умолчанию 3100)
   - **Node Version:** рекомендуется Node.js 14+ или 18+

### Примеры для разных платформ:

#### Render.com
1. Создайте новый Web Service
2. Подключите репозиторий GitHub
3. Настройки:
   - **Build Command:** `npm install`
   - **Start Command:** `npm start`
   - **Environment:** Node

#### Railway.app
1. Создайте новый проект
2. Подключите репозиторий GitHub
3. Railway автоматически определит Node.js проект
4. Убедитесь, что в `package.json` есть скрипт `start`

#### Fly.io
1. Установите Fly CLI
2. Выполните `fly launch` в папке проекта
3. Fly автоматически создаст конфигурацию

### Важные замечания:

- Файлы `gantt-state.json` и `company-info.json` создаются автоматически при первом использовании
- Данные сохраняются на сервере в файловой системе
- Для продакшена рекомендуется использовать базу данных вместо файлов

### Переменные окружения:

- `PORT` - порт для запуска сервера (по умолчанию 3100)

### Структура API:

- `GET /api/gantt-state` - получить сохраненное состояние диаграммы
- `POST /api/gantt-state` - сохранить состояние диаграммы
- `GET /api/company-info` - получить информацию о компании
- `POST /api/company-info` - сохранить информацию о компании

### Проверка работы:

После деплоя откройте:
```
https://your-domain.com/implementation_schedule.html
```

Убедитесь, что:
- Страница загружается
- Можно создавать и редактировать задачи
- Данные сохраняются (проверьте через API)
